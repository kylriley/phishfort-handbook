# GitHub Takedowns

## Evidence Requirements

✅ **Type of Attack**
- Specify: phishing, credential exposure, or other brand infringement.

✅ **File Containing the Infringement**
- Provide exact file name.

✅ **Location of Evidence**
- Repository path and file location.
- Example: `username/repository-name/folder/file.ext`.

✅ **Lines of Code** (for partial infringement)
- Specify exact lines containing the infringement.
- Not needed if entire file/repository is infringing.

✅ **Context/Incident Explanation**
- Explain why the content infringes on your brand or intellectual property.

## Best Practices

✅ Be specific about what's infringing.
✅ Provide repository URL.
✅ Specify exact files and lines when possible.
✅ Explain the harm or violation.

❌ Don't report entire repositories without specifying infringing parts.
❌ Don't submit without clear explanation of infringement.
