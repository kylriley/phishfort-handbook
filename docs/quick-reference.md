# Quick Reference: Case Evidence Requirements

This table provides a quick summary of evidence requirements. For comprehensive information and detailed guidance on each case type, please refer to the relevant sections throughout this handbook.

| Case | Evidence Required | Comments |
|------|------------------|----------|
| **Website impersonation** | 1. Original website link{% br /%}2. Statements of abuse{% br /%}3. Screenshots showing malicious targeting. | Site must be maliciously targeting the brand in question. |
| **Social Media Accounts** | 1. Legitimate account as reference{% br /%}2. Letter of Authorization{% br /%}3. Statement of Impersonation{% br /%}4. 2+ from: impersonating logo/name/post with affiliation claim. | Requires minimum of 2 impersonation criteria to be met. |
| **Messaging Platforms** | 1. Platform/messenger name{% br /%}2. Username{% br /%}3. Date/timestamp{% br /%}4. Screenshots of infringing text{% br /%}5. Summary of malicious activity. | Submit as URL, screenshot, or text where visible. |
| **Phone scams** | 1. Caller number{% br /%}2. Receiver number{% br /%}3. Timezone{% br /%}4. Description of conversation{% br /%}5. SMS text (if applicable). | Description should include impersonation statement. |
| **TM/Copyright infringement** | 1. Proof of ownership (TM registration, copyright docs){% br /%}2. Evidence of unfair use{% br /%}3. Letter of Authorization. | LoA must match the trademark registration. |
| **APK websites** | 1. Link to original app{% br /%}2. Screenshots of malicious APK{% br /%}3. Evidence of brand misuse. | - |
| **Email attack** | 1. Email headers (raw/original){% br /%}2. Malicious or impersonating content{% br /%}3. Screenshots of email. | Headers must be from original email, not forwarded. |
| **Similar/Typo squat domain** | 1. Domain name{% br /%}2. Screenshots of content{% br /%}3. Evidence of malicious use. | Typosquat alone is not enough. Higher success if domain has malformed/improper wording (not generic words). |
| **Parked page** | 1. Clear trademark in link{% br /%}2. Evidence TM is not generic. | No legal basis to take down parked pages alone. Attempts performed only when possible. |
| **Redirect source** | 1. Source domain with clear TM{% br /%}2. Target domain details. | Submit target domain as separate incident with comment about redirect. |
| **Scams** | 1. Service promised (documentation){% br /%}2. Proof of transaction{% br /%}3. Proof of non-delivery. | Case-specific, usually requires court-level evidence. |
| **Suspicious / not malicious** | 1. Context and details. | Research possible as part of detection package. |
| **Other** | 1. Detailed reproduction steps{% br /%}2. Description of scam/threat{% br /%}3. Any available evidence. | Explain in detail how to reproduce the case. |
